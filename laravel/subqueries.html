Using Subqueries to Get a Computed Column

Add a last_login_at column to each user with a single query.

You can add a computed column last_login_at to each user using a subquery:

$users = User::query()
    ->addSelect(['last_login_at' => Login::select('created_at')
        ->whereColumn('user_id', 'users.id')
        ->orderByDesc('created_at')
        ->limit(1)
    ])
    ->get();

This adds a computed column last_login_at to each user. Update the Blade view to display it:

{{ $user->last_login_at }}

Note that we no longer have the diffForHumans formatting because this is now a string, not a Carbon instance.

Fixing Date Formatting with Query-Time Casting

Adding diffForHumans() to last_login_at causes an error because it's a string, not a Carbon instance. Laravel solves this with query-time casting:

$users = User::query()
    ->addSelect(['last_login_at' => Login::select('created_at')
        ->whereColumn('user_id', 'users.id')
        ->orderByDesc('created_at')
        ->limit(1)
    ])
    ->withCasts(['last_login_at' => 'datetime'])
    ->get();

Now diffForHumans() works again in the Blade view:

{{ $user->last_login_at->diffForHumans() }}
