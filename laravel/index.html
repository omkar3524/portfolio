Using Subqueries to Get the Last Login Date

You can add a computed column last_login_at to each user using a subquery: 

$users = User::query()
                ->addSelect(['last_login_at' => Login::select('created_at')
                ->whereColumn('user_id', 'users.id')
                ->orderByDesc('created_at')
                ->limit(1)
                ])
                ->get();

This adds a computed column last_login_at to each user.

Update the Blade view to display this new attribute: 

{{ $user->last_login_at }}

Note that we no longer have the diffForHumans formatting because this is now a string, not a Carbon instance.

Fixing Date Formatting with Query-Time Casting

Back in the Blade view, adding diffForHumans() to last_login_at causes an error because it's a string, not a Carbon instance.

Previously, this worked because the login model cast the created_at attribute to Carbon automatically.

Now that we're getting the date directly in the users query, this casting doesn't happen.

Laravel provides an elegant solution called query-time casting.

Update the query to add a cast: 

$users = User::query()
                ->addSelect(['last_login_at' => Login::select('created_at')
                ->whereColumn('user_id', 'users.id')
                ->orderByDesc('created_at')
                ->limit(1)
                ])
                ->withCasts(['last_login_at' => 'datetime'])
                ->get();

Now, in the Blade view, diffForHumans() works again: 

{{ $user->last_login_at->diffForHumans() }}

Dynamic Relationships in Laravel

We can achieve this using dynamic relationships.

Step 1: Update the User Model

  1. Remove the two last login scopes in the User model.
  2. Add a new lastLogin belongsTo relationship:
public function lastLogin()
                {
                return $this->belongsTo(Login::class, 'last_login_id');
                }

Normally, for a belongsTo relationship to work, the users table needs a foreign key columnâ€”in this case, last_login_id. Since our users table does not have this column, we'll use a subquery to select it instead.

Step 2: Create a withLastLogin Scope

public function scopeWithLastLogin($query)
                {
                $query->addSelect(['last_login_id' => Login::select('id')
                ->whereColumn('user_id', 'users.id')
                ->orderByDesc('created_at')
                ->limit(1)
                ])->with('lastLogin');
                }

This adds a computed last_login_id column and eager loads the lastLogin relationship automatically. Eloquent treats this computed column as if it were a real column, so everything works seamlessly.

Step 3: Update the Query in UserController

$users = User::withLastLogin()->paginate();

In the Blade view, instead of using the last_login_at attribute, use the lastLogin relationship to access properties:

{{ $user->lastLogin->created_at->diffForHumans() }}
                {{ $user->lastLogin->ip_address }}

Refreshing the browser shows everything works as expected.

Important Note on Lazy Loading:
You cannot lazy load dynamic relationships because unless you explicitly call the withLastLogin scope, the last_login_id won't be present on the model. This isn't usually an issue since dynamic relationships are typically used to solve performance problems where lazy loading is not ideal.
