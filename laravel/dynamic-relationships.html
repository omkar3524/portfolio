Dynamic Relationships in Laravel

Eager-load a relationship that has no real foreign key column.

Step 1: Update the User Model

  1. Remove the two last-login scopes in the User model.
  2. Add a new lastLogin belongsTo relationship:
public function lastLogin()
{
    return $this->belongsTo(Login::class, 'last_login_id');
}

Normally a belongsTo needs a foreign key column (last_login_id). Since our users table doesn't have one, we use a subquery to select it.

Step 2: Create a withLastLogin Scope

public function scopeWithLastLogin($query)
{
    $query->addSelect(['last_login_id' => Login::select('id')
        ->whereColumn('user_id', 'users.id')
        ->orderByDesc('created_at')
        ->limit(1)
    ])->with('lastLogin');
}

This adds a computed last_login_id column and eager-loads the lastLogin relationship. Eloquent treats the computed column as if it were real, so everything works seamlessly.

Step 3: Update the Query in UserController

$users = User::withLastLogin()->paginate();

In Blade, use the relationship instead of the attribute:

{{ $user->lastLogin->created_at->diffForHumans() }}
{{ $user->lastLogin->ip_address }}
⚠️ Note on Lazy Loading

You cannot lazy-load dynamic relationships, because unless you explicitly call the withLastLogin scope, the last_login_id won't be present on the model. This is rarely an issue, since dynamic relationships are typically used to solve performance problems where lazy loading isn't ideal.
